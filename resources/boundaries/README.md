# Boundaries

Service interfaces and integration points documented from analyzed repositories.

## What Goes Here

- Service API contracts
- Module boundaries and interfaces  
- External integration patterns
- Cross-service communication patterns

## File Format

Each boundary file should include:

- **Source:** Which repository it was extracted from
- **Triggers:** Keywords for prompt-router detection
- **Interface:** API contracts with code references
- **Dependencies & Consumers:** What interacts with this boundary

See `goals/repository-learning.md` for the complete template.
